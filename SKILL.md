# Judgment Anchoring

本 skill 是一个**概念锚点执行器**。唯一职责:

1. 从锚点库路由到匹配的 domain
2. 从 domain 抽取锚点 + 必要规则，组装成提示词
3. 执行这份提示词

**锚点** = 具体人名 + 一句 facet hint。通过召唤模型对权威作品的训练留痕，在任务中激活判断力。不是规则，不是抽象原则。

**必要规则** = 锚点无法自带、但执行时必须遵守的硬约束(语言、工具、输出格式、该领域的客观陷阱)。每个 domain 都有，长短因领域而异。

## 起手

执行任务时,**第一步读取 `flows/execute.md`**。其 Step 0 会引导你查 `references/anchor-catalog.md` 路由到目标 domain。不允许凭记忆或推理跳过流程。

## 意图识别

| 意图信号                                       | 分支 | 文件                |
|--------------------------------------------|----|-------------------|
| 写代码、设计 API、做架构决策、写技术文档、创作、思辨讨论 / 决策取舍类对话 等 | 执行 | flows/execute.md  |
| 回写新配方、评估或修订已有配方、新建领域                       | 维护 | flows/maintain.md |

意图不明确时倾向"执行"分支。
