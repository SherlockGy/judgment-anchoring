# 求证：先证伪、先看新

**用途：** 锚点能产出判断（假设），但出去核对现实不是锚点能做的事——这一步独立成文。判断密集、且结论依赖对现实认识的 domain（思辨 / 决策 / 取舍 / 诊断等）在产出判断后引用本文件做求证；纯价值取舍、写代码、写小说等无可核实事实主张的产出不需要。

**位置固定：在锚点产出判断之后。** 不能边想边查（污染判断的大胆），也不能先查再想（被现成信号先入为主带偏）。委派 / 自动执行时写进任务描述随 prompt 带走（不进锚点表）。

## 怎么做

锚点产出的只是假设。把判断里**依赖事实**的部分对照现实核验，方向焊死：

- **先证伪**：假设结论可能已过时或有偏，主动找最权威的**相反**信号，不搜佐证；搜到反证正面纳入校准，不视而不见。
- **先看新**：按新近度排序、越新越优先，尤其近几个月、与主流叙事不符的动向（快速变化的领域里，最新的反向信号最可能推翻来自训练分布的旧叙事）。
- 查什么按问题类型：代码 / 文献 / 新闻 / 社区 / 数据。

否则求证退化成自我确认——顺着先验搜佐证，把训练云旧叙事又捞一遍。有工具就查；没工具就明列"哪几点需核实、怎么核实"，不把未核实的假设当结论交付。

这是无切身经历的系统换可靠性的方式：补不了亲历的重量，就靠对现实的核对——但只有朝证伪和最新用力才算数。
